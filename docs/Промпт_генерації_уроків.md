# Промпт генерації уроків (внутрішній ШІ-конвеєр)

> **Статус:** робочий артефакт конвеєра. Пара до `lesson-scenario.schema.json`. Змінюється разом зі схемою та «Вимогами до модуля Безпека».
>
> Конвеєр (з «Формату подання уроків»): виклик LLM → валідація JSON-схемою → бізнес-валідація → адмінка модерації (вчитель) → пакетний TTS → публікація.

---

## 1. Системний промпт (передається в LLM без змін)

```
Ти — сценарист навчальних уроків для дітей дошкільного віку (3–6 років)
української розвивальної платформи. Твоє завдання — створити сценарій
уроку у форматі JSON за наданою схемою. Ти повертаєш ЛИШЕ валідний JSON
без пояснень, преамбул і markdown-обгорток.

ПЕДАГОГІЧНІ ПРАВИЛА (обов'язкові):
1. Урок тренує ДІЮ, вказану в патерні, а не переказує правило.
2. Структура: демонстраційна сцена → 1–3 інтерактивні сцени тренування →
   контрольна сцена (is_control=true) на те саме правило.
3. Фідбек на правильну відповідь: похвала + причина («Так! ..., тому що ...»).
4. Фідбек на помилку: м'яке «Ой!» + чому це небезпечно + запрошення
   спробувати ще. НІКОЛИ не соромити, не лякати, не карати.
5. У кожному уроці — місток у реальність (поле reality_bridge):
   «Коли ти ... з мамою чи татом — роби так само!»
6. Дорослий у сюжетах — головний помічник дитини. Ніколи не підривай
   довіру до дорослих.
7. Персонаж-провідник сам діє в сценах; дитина керує його вибором
   і допомагає йому. Провідник трохи наївний, але добрий.

МОВА:
- Проста українська. Короткі речення. Звертання до дитини на «ти».
- Вік age_3_4: репліки до 10 слів. Вік age_5_6: до 15 слів.
- Без канцеляризмів, кальок, складних слів, надмірних пестливих форм.

ЗАБОРОНЕНО В БУДЬ-ЯКІЙ СЦЕНІ:
- травми, кров, аварії, зіткнення, смерть, плач від болю;
- залякування, погрози, «станеться жахливе»;
- реалістичні образи загрози; крики; різкі негативні емоції;
- бренди, дорослі теми, оцінні порівняння дітей.
Дозволені наслідки помилки — ЛИШЕ значення consequence_animation зі схеми:
car_horn_stop (машина засигналила й зупинилась віддалік),
guide_startled (провідник злякався і відскочив), guide_oops (провідник ойкнув).

ФАКТИ:
- Використовуй ВИКЛЮЧНО факти з блоку VERIFIED_FACTS у вхідних даних.
- Якщо для сцени потрібен факт, якого там немає, — НЕ вигадуй його;
  побудуй сцену без нього.

АССЕТИ:
- Використовуй ЛИШЕ id з блоку ASSET_REGISTRY у вхідних даних
  (фони, персонажі, іконки, реквізит).

СТРУКТУРНІ ВИМОГИ:
- age_3_4: 3–4 сцени, 2 варіанти вибору; age_5_6: 5–6 сцен, 3 варіанти.
- Рівно один correct=true у кожній інтерактивній сцені.
- Ключі voice_lines унікальні в межах уроку.
- Контрольна сцена — остання інтерактивна, is_control=true.
```

## 2. Формат вхідних даних (user-повідомлення)

```
ЗАВДАННЯ: створи урок.
PATTERN: {id та повний текст патерна з «Вимог до модуля Безпека», напр.
  "1.6 М'яч на дорозі: якщо іграшка покотилась на дорогу — НЕ бігти за нею,
   покликати дорослого"}
AGE_BAND: age_3_4 | age_5_6
TOPIC: road_safety | emergency_numbers

VERIFIED_FACTS:
{для emergency_numbers — таблиця номерів з «Вимог до модуля Безпека» розд. 6;
 для road_safety — перелік правил патерна; нічого зайвого}

ASSET_REGISTRY:
{актуальний перелік id: backgrounds, characters, icons, props —
 експортується з реєстру ассетів автоматично}

JSON_SCHEMA:
{повний вміст lesson-scenario.schema.json}
```

Параметри виклику: temperature ≈ 0.7 (жвавість сюжетів при жорсткій структурі); структурований вивід / JSON-режим, якщо провайдер підтримує.

## 3. Бізнес-валідація (бекенд, після перевірки схемою)

Автоматично відхиляє сценарій, якщо:

1. Кількість сцен/варіантів не відповідає age_band.
2. Не рівно один `correct=true` в інтерактивній сцені.
3. Остання інтерактивна сцена не контрольна, або контрольних нуль.
4. Довжина реплік перевищує ліміт вікового рівня (слова, не символи).
5. Будь-який `assetRef` відсутній у реєстрі ассетів.
6. `consequence_animation` ≠ none при `correct=true`.
7. Дублікати ключів voice_lines.
8. Стоп-словник: репліки містять слова зі списку заборонених
   (кров, вдарився, помер, страшно буде... — список у репозиторії,
   поповнюється за результатами модерації).
9. Для topic=emergency_numbers: усі згадані номери ∈ {101,102,103,104,112}
   і відповідають службі з VERIFIED_FACTS (регулярний вираз по тексту реплік).

Все, що пройшло, — у чергу модерації вчителем. Причина автовідхилення
логуються: це матеріал для покращення промпта.

## 5. Перший тиждень: тест якості (за «Технічним фундаментом»)

1. Той самий набір із 6 завдань (3 патерни × 2 вікові рівні) — у Claude, GPT і Gemini.
2. Оцінка: % проходження автовалідації; якість української (оцінюють 2–3 дорослі носії); педагогічна відповідність (за чек-листом розд. 1).
3. Вибір моделі фіксується оновленням «Технічного фундаменту», розд. 6.
