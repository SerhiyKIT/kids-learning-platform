# Модель даних

> **Статус:** зафіксоване рішення технічного фундаменту. Ґрунтується на «Баченні проекту» та «Форматі подання уроків та ШІ».
>
> **Зафіксовані вибори:** база даних — **PostgreSQL** (реляційні таблиці + JSONB для сценаріїв уроків); акаунт дитини створюють **і батьки, і вчитель** (два потоки); зберігаємо **кожну відповідь дитини в кожній сцені** (без надлишкової телеметрії).

---

## 1. Принципи

1. **Строгість у чутливому, гнучкість у контенті.** Користувачі, ролі, згоди, результати — чіткі реляційні таблиці. Сценарій уроку — JSONB-документ за схемою сцен-шаблонів.
2. **Мінімізація даних дитини.** Про дитину зберігаємо лише необхідне: ім'я/псевдонім, рік народження (не повна дата), навчальні результати. Жодних фото, адрес, аудіозаписів голосу.
3. **Кожен запис про дитину має «власника-дорослого».** Дані дитини завжди прив'язані до батьківської згоди; відкликання згоди → каскадне видалення.
4. **Версіонування контенту.** Урок ніколи не редагується «в місці» після публікації — створюється нова версія. Результати дитини завжди посилаються на конкретну версію.
5. **Аудіо — кешований артефакт.** Озвучення зберігається окремо від уроків і перевикористовується за хешем фрази.

## 2. Огляд сутностей

```
users (дорослі) ──┬── parent_child ──── children ──── group_members ──── groups ──── users(вчитель)
                  │                        │                                │
consents ─────────┘                        │                          lesson_assignments
                                           │                                │
child_style_prefs ── themes          lesson_attempts ──── scene_answers     │
                                           │                                │
assistant_dialogs                          └──────── lesson_versions ── lessons ── modules
                                                          │
invitations                                          audio_assets
```

## 3. Користувачі та доступ

### users — дорослі користувачі (батьки, вчителі, адміністратори)
| Поле | Тип | Примітка |
|---|---|---|
| id | uuid PK | |
| email | text unique | логін |
| password_hash | text | або зовнішній auth-провайдер |
| role | enum: parent, teacher, admin | одна роль; сумісництво = два акаунти або окрема таблиця ролей у майбутньому |
| display_name | text | |
| locale | text | 'uk' за замовчуванням |
| created_at, updated_at | timestamptz | |
| deleted_at | timestamptz null | м'яке видалення |

### children — профілі дітей
Дитина **не має email/пароля**. Вхід — через пристрій дорослого або дитячий вхід у групі (вибір свого аватара + простий піктограмний код).

| Поле | Тип | Примітка |
|---|---|---|
| id | uuid PK | |
| display_name | text | ім'я або псевдонім — мінімізація |
| birth_year | smallint | лише рік, для вікової адаптації контенту |
| avatar_id | text | з бібліотеки аватарів, не фото |
| pin_code_hash | text null | піктограмний код для самостійного входу |
| created_by | uuid FK→users | хто створив профіль (батько або вчитель) |
| status | enum: pending_consent, active, suspended, deleted | **pending_consent** — створений вчителем, чекає згоди батьків |
| created_at, deleted_at | timestamptz | |

### parent_child — зв'язок батьків і дітей (багато-до-багатьох)
| Поле | Тип | Примітка |
|---|---|---|
| parent_id | uuid FK→users | role=parent |
| child_id | uuid FK→children | |
| relation | enum: mother, father, guardian | |
| is_primary | boolean | головний контакт для згод |

### consents — згоди батьків
Окремий облік кожної згоди — юридична основа обробки даних дитини.

| Поле | Тип | Примітка |
|---|---|---|
| id | uuid PK | |
| child_id | uuid FK→children | |
| parent_id | uuid FK→users | хто надав |
| type | enum: account, microphone, assistant_level2, style_personalization | перелік розширюваний |
| granted_at | timestamptz | |
| revoked_at | timestamptz null | відкликання → відповідна функція вимикається негайно |

**Правило двох потоків:** якщо профіль створили батьки — згода `account` фіксується при створенні. Якщо створив вчитель — профіль у статусі `pending_consent`, вчитель надсилає запрошення (див. invitations), батько приєднується і надає згоду; до цього дитина може проходити уроки, але **дані не накопичуються довше пробного періоду** (політика уточнюється в документі «Приватність»).

## 5. Контент

### modules — освітні модулі
Довідник: безпека, математика, логіка, читання, письмо. `id, code, name, sort_order`.

### lessons — уроки (контейнер)
| Поле | Тип | Примітка |
|---|---|---|
| id | uuid PK | |
| module_id | FK→modules | |
| owner_teacher_id | uuid FK→users null | null = загальний каталог платформи |
| title | text | |
| current_version_id | uuid FK→lesson_versions | опублікована версія |
| created_at, deleted_at | | |

### lesson_versions — версії уроку
| Поле | Тип | Примітка |
|---|---|---|
| id | uuid PK | |
| lesson_id | FK→lessons | |
| version_no | int | |
| scenario | **jsonb** | повний JSON-сценарій: сцени за шаблонами, репліки помічника (підказки/пояснення/фідбек), посилання на ассети й аудіо |
| generated_by | enum: ai, human, ai_edited | |
| ai_model | text null | напр. 'claude-…', для відтворюваності |
| status | enum: draft, generated, auto_validated, rejected_auto, approved, published, archived | конвеєр модерації з «Формату подання уроків» |
| approved_by | uuid FK→users null | вчитель, що затвердив |
| created_at | timestamptz | |

**Схема `scenario` (скорочено):**
```json
{
  "learning_goal": "…",
  "scenes": [{
    "type": "choice_situation",        // один із 6 затверджених шаблонів
    "background": "street_day",         // id з бібліотеки ассетів
    "characters": [{"id":"fox","emotion":"neutral"}],
    "voice_lines": [{"key":"intro","text":"…"}],
    "options": [{"id":"a","label_icon":"traffic_light","correct":true,
                 "feedback":{"text":"Правильно, тому що…","voice_key":"fb_a"}}],
    "assistant": {
      "hints": [{"level":1,"text":"…"},{"level":2,"text":"…"}],
      "re_explain": [{"text":"…"}]
    }
  }]
}
```

### audio_assets — озвучення (кеш TTS)
| Поле | Тип | Примітка |
|---|---|---|
| id | uuid PK | |
| text_hash | text unique | sha256(нормалізований текст + voice + provider) — дедуплікація |
| text | text | вихідна фраза |
| provider, voice_id | text | напр. 'elevenlabs' / id голосу |
| file_url | text | об'єктне сховище (S3-сумісне) |
| duration_ms | int | |

### themes / child_style_prefs — персоналізація стилю
- `themes`: затверджені теми оформлення бібліотеки ассетів (`id, name, palette jsonb, guide_character, status: draft/approved`). Згенеровані ШІ теми проходять модерацію так само, як уроки.
- `child_style_prefs`: `child_id, answers jsonb (уподобання), theme_id FK` — потребує згоди `style_personalization`.

## 7. Права доступу до даних (стисло; деталі — в документі «Ролі та приватність»)

| Дані | Дитина | Батьки | Вчитель | Адмін |
|---|---|---|---|---|
| Свій профіль дитини | бачить аватар/тему | повний | ім'я + результати у своїх групах | повний |
| Результати (attempts, answers) | свій прогрес у ігровій формі | своїх дітей | дітей своїх груп | повний |
| Журнал помічника | — | своїх дітей | дітей своїх груп | повний |
| Уроки | опубліковані призначені | перегляд призначених | свої + каталог, CRUD своїх | усі |
| Згоди | — | керує | бачить статус | повний |

---

## 8. Видалення даних

- Відкликання згоди `account` або запит батьків → **каскадне видалення** children → attempts, scene_answers, assistant_dialogs, style_prefs, parent_child, group_members, consents.
- Агреговану неперсоніфіковану статистику (без child_id) можна зберігати для покращення уроків.
- `audio_assets` не містять даних дітей — не каскадуються.

---

## 9. Рішення, зафіксовані цим документом

1. **PostgreSQL**: реляційне ядро + JSONB для сценаріїв уроків.
2. **Два потоки створення профілю дитини** (батьки / вчитель) через `invitations` + статус `pending_consent`; у MVP допустимо ввімкнути спершу один потік — модель готова до обох.
3. **Гранулярність результатів — кожна відповідь у сцені** (`scene_answers`), без надлишкової телеметрії.
4. Згоди батьків — **окрема таблиця `consents` за типами**, з відкликанням і каскадним видаленням.
5. Результати завжди прив'язані до **конкретної версії уроку** (`lesson_versions`).
6. Озвучення — **дедуплікований кеш `audio_assets`** за хешем фрази.
