# Обсяг MVP (перша версія)

> **Статус:** зафіксоване рішення. Ґрунтується на «Баченні проекту», «Форматі подання уроків та ШІ» і «Моделі даних».
>
> **Зафіксовані вибори:** модуль — **лише безпека**; ШІ-генерація — **внутрішній інструмент** (без інтерфейсу для вчителів); функції реального часу — **без голосу, помічник лише скриптований**.
>
> **Мета MVP:** якнайшвидше перевірити найризиковіші гіпотези проекту на реальних дітях, побудувавши мінімум.

---

## 1. Гіпотези, які перевіряє MVP

1. **Педагогічна:** діти 3–6 років проходять інтерактивні ситуації із задоволенням і після кількох повторень відтворюють правильний патерн (перевіряється на модулі безпеки — найвимогливішому).
2. **Формат:** сцени-шаблони + скриптований помічник (підказки, пояснення, фідбек «правильно, тому що…») достатні для навчання без живого дорослого поруч.
3. **ШІ-контент:** LLM генерує якісні україномовні сценарії за схемою, які вчитель затверджує з мінімальними правками.
4. **Технічна:** 2D-рушій + кешоване аудіо працюють плавно на слабких планшетах/телефонах.

---

## 2. Що ВХОДИТЬ у MVP

### Контент
- **Один модуль: Безпека.** Дві теми на старт:
  - **Правила дорожнього руху** (перехід дороги, світлофор, поведінка біля дороги);
  - **Номери екстрених служб** (101/102/103/104, 112 — який номер у якій ситуації, що сказати).
- Орієнтир обсягу: **8–12 уроків** по 4–6 сцен (сесія дитини ~5–7 хвилин).
- **Одна тема оформлення** (без персоналізації стилів) та **один персонаж-провідник**.

### Сцени-шаблони (4 із 6 затверджених)
1. Демонстраційна сценка (пояснення перед інтерактивом);
2. Ситуація з вибором (ядро модуля безпеки);
3. Сортування «безпечно / небезпечно»;
4. Діалог із запитанням — відповідь **дотиком** (голосова відповідь відкладена).

«Знайди правильне» і «Повтори за персонажем» — після MVP (потрібні модулям читання/письма/математики).

### Помічник (лише рівень 1 — скриптований)
- Східчасті підказки (2 рівні), повторне пояснення на запит («кнопка-персонаж»), похвала, м'яка корекція, місток у реальність.
- Усі репліки генеруються разом з уроком, модеруються і озвучуються пакетно.

### ШІ-конвеєр (внутрішній)
- Генерація JSON-сценаріїв через LLM API запускається **командою/простою адмін-сторінкою** — лише для команди проекту.
- Автовалідація за схемою → перегляд і затвердження у **мінімальній адмінці** (перегляд сценарію, правка тексту, кнопки затвердити/відхилити) → пакетний TTS → публікація.
- Вибір LLM і голосу TTS — за процедурами з «Формату подання уроків» (тест української, сліпий тест голосів) **на початку розробки MVP**.

### Ролі та кабінети
- **Дитина:** вибір аватара, список призначених уроків, проходження, простий ігровий прогрес (зірочки/наліпки).
- **Вчитель:** створення групи, код приєднання, призначення уроків **із готового каталогу** групі, перегляд результатів дітей своїх груп (по сценах, з підказками).
- **Батьки:** створення профілю дитини, приєднання до групи за кодом, згоди, історія всіх уроків і відповідей дитини.
- **Реєстрація дитини в MVP — лише через батьків** (потік «вчитель створює» відкладений; модель даних уже готова до нього).
- **Адмін:** модерація і публікація контенту (мінімальна адмінка вище).

### Платформа
- **PWA**: телефон, планшет, десктоп; встановлення на домашній екран.
- **Кешування уроків і аудіо** — призначений урок працює при поганому інтернеті.
- Керування — **лише дотик**.

## 3. Що НЕ входить у MVP (свідомо відкладено)

| Відкладено | Коли повертаємось |
|---|---|
| Модулі: математика, логіка, читання, письмо | після підтвердження гіпотез 1–2 |
| Голосове керування (STT) | окремий **прототип-дослідження паралельно з MVP** (див. розд. 5), у продукт — після нього |
| Помічник рівень 2 (вільні запитання, LLM у реальному часі) | після голосу; потребує фільтрів і батьківського перемикача |
| Інтерфейс ШІ-генерації для вчителів | після стабілізації якості генерації на внутрішньому конвеєрі |
| Персоналізація стилів дитиною | після MVP; потребує кількох тем ассетів |
| Потік «вчитель створює профіль дитини» + запрошення батьків | після MVP |
| Редактор уроків для вчителя (ручне створення/правка) | разом з інтерфейсом генерації |
| Сповіщення, звіти на email, експорт | за потребою |

**Правило:** якщо під час розробки з'являється ідея «а давай ще…» — вона записується в беклог, а не в MVP. Розширення MVP — лише свідомим оновленням цього документа.

## 4. Критерії готовності та успіху MVP

**Готовність до тесту з дітьми:**
- 8+ опублікованих уроків безпеки, повністю озвучених;
- проходження працює на бюджетному Android-планшеті без «гальм»;
- батьки бачать історію відповідей; вчитель — результати групи.

**Успіх (перевірка гіпотез):**
- дитина самостійно проходить урок до кінця без допомоги дорослого в ≥70% випадків;
- після 2–3 повторень теми дитина правильно відповідає на контрольні сцени (напр., обирає правильний номер служби) у ≥80% випадків;
- ≥70% згенерованих ШІ сценаріїв затверджуються вчителем без структурних переробок (лише дрібні правки тексту);
- урок з аудіо завантажується і працює офлайн після першого відкриття.

---

## 5. Паралельний трек: прототип голосу (не блокує MVP)

Найризиковіша технологія — розпізнавання дитячого українського мовлення. Щоб не гальмувати MVP і не закладати неперевірених припущень:

- окремий міні-прототип: Web Speech API проти хмарного STT на 10–15 типових дитячих відповідях («сто один», «зебра», «так/ні»);
- тест із 3–5 дітьми цільового віку;
- результат — рішення: який STT, які типи відповідей реалістичні → вхідні дані для версії 2.

---

## 6. Орієнтовна послідовність розробки MVP

1. Технічний фундамент: PWA-каркас, БД за моделлю даних, автентифікація дорослих.
2. Рушій сцен: 4 шаблони + програвання аудіо + кешування.
3. Бібліотека ассетів v1: персонаж-провідник, 2 фони (вулиця, дім), базові об'єкти для двох тем.
4. Внутрішній ШІ-конвеєр: промпт + схема + автовалідація + адмінка модерації + пакетний TTS.
5. Кабінети: батьки → вчитель → дитячий інтерфейс.
6. Контент: генерація і затвердження 8–12 уроків.
7. Тест з дітьми → вимірювання критеріїв успіху → рішення про версію 2.

---

## 7. Рішення, зафіксовані цим документом

1. MVP = **модуль «Безпека»** (ПДР + номери служб), 8–12 уроків, 4 сцени-шаблони, одна тема оформлення.
2. ШІ-генерація — **внутрішній інструмент команди**; вчителі в MVP користуються готовим каталогом.
3. **Без голосу**; помічник — **лише скриптований рівень 1**; голос досліджується паралельним прототипом.
4. Реєстрація дитини в MVP — **лише через батьків**.
5. Розширення обсягу MVP — лише через свідоме оновлення цього документа.
