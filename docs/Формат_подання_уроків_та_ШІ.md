# Формат подання уроків та вимоги до ШІ

> **Статус:** зафіксоване рішення технічного фундаменту. Доповнює «Бачення проекту». Змінюється лише свідомим переглядом цього документа.

---

## 1. Базове рішення

**2D-векторна графіка + повне озвучення + інтерактивні сцени-шаблони, які ШІ наповнює структурованими даними, а не малює.**

Обґрунтування (за трьома критеріями проекту):

| Критерій | 2D + шаблони | 3D |
|---|---|---|
| Оптимізація / всі пристрої | Легкі векторні ассети, кешування, працює на слабких планшетах і поганому інтернеті | Потребує GPU, великі ассети, довге завантаження |
| Придатність для генерації ШІ | LLM чудово генерує структуровані сценарії уроків | ШІ не може якісно створювати 3D-сцени (моделі, ріг, анімація) |
| Сприйняття дошкільнятами | Прості, чисті, впізнавані образи — оптимально для 3–6 років | Зайвий візуальний шум без педагогічної переваги |

## 2. Подання інформації дитині

### 2.1. Озвучення — головний канал

- Діти цільового віку здебільшого **не читають** → 100% навчального змісту доноситься **голосом**: інструкції, пояснення, фідбек.
- Текст на екрані допустимий лише як **предмет вивчення** (літера, слово в модулі читання) або дубляж для дорослих — ніколи як єдиний носій інструкції.
- Кожна фраза озвучення коротка (1–2 речення), проста лексика, повільний темп.

### 2.2. Анімація показує — голос пояснює

- Анімована сценка демонструє **дію** (персонаж зупиняється перед дорогою), голос дає **пояснення й правило**.
- Не пасивний мультик, а **інтерактивна ситуація**: сценка ставить дитину перед вибором → дитина діє → сценка програє наслідок → голос дає фідбек («Правильно, **тому що**…»).
- Для модуля безпеки це основний формат: тренування патерну дії, а не переказ правила.

### 2.3. Керування

- Дотик (основне) + **голосове керування** (відповіді голосом, прості команди).
- Великі зони натискання, жодних дрібних елементів, жодного читання для навігації (іконки + озвучення при наведенні/першому показі).

## 3. Сцени-шаблони (конструктор уроків)

Урок = послідовність сцен. Кожна сцена — один із затверджених типів шаблонів:

1. **Ситуація з вибором** — персонаж у ситуації, 2–3 варіанти дії, програвання наслідку, фідбек. (Основа модуля безпеки.)
2. **Сортування / групування** — перетягнути об'єкти в правильні групи. (Логіка, математика, безпека: «безпечно / небезпечно».)
3. **Знайди правильне** — вибрати серед об'єктів на сцені. (Математика, читання: «знайди літеру А».)
4. **Повтори за персонажем** — обвести, накреслити, вимовити. (Письмо, читання.)
5. **Діалог із запитанням** — персонаж запитує, дитина відповідає дотиком або голосом. (Усі модулі; безпека: «який номер викликати?».)
6. **Демонстраційна сценка** — коротка анімація-пояснення перед інтерактивом (до 30 секунд).

Правила:
- Нові типи шаблонів додаються лише через оновлення цього документа й розробку рушія — **ШІ не вигадує нових механік.**
- Усі візуальні елементи — з **затвердженої бібліотеки ассетів** (персонажі з наборами емоцій/поз, фони, об'єкти, готові анімації).
- Персоналізація стилю під уподобання дитини = **теми оформлення** тієї самої бібліотеки (палітри, персонаж-провідник, декор), а не генерація графіки з нуля.

## 4. ШІ для створення уроків

### 4.1. Тип ШІ

**LLM (велика мовна модель) через API з режимом структурованого виводу.** ШІ генерує урок як **JSON-сценарій**: тип сцени, фон, персонажі, репліки для озвучення, варіанти вибору, фідбек на кожну відповідь, навчальна ціль сцени.

Основні кандидати (рівноцінні за придатністю, вибір за тестом якості української та ціною):
- **Anthropic Claude API** — сильна робота з довгими структурованими промптами, tool use / структурований вивід.
- **OpenAI API (GPT)** — режим structured outputs із гарантованою JSON-схемою.
- **Google Gemini API** — структурований вивід, конкурентна ціна.

### 4.2. Вимоги до генерації (обов'язкові)

1. **Строга JSON-схема.** Вивід валідується проти схеми сцен-шаблонів; невалідний результат відхиляється автоматично, не потрапляючи до вчителя.
2. **Якісна українська.** Проста лексика для 3–6 років, без кальок і канцеляризмів. Обов'язкове тестування якості мови на етапі вибору моделі.
3. **Системний промпт із педагогічними правилами** проекту: навчання первинне, небезпека без залякування, фідбек «правильно, тому що…», місток у реальність.
4. **Фактична безпека.** Для модуля безпеки ШІ отримує перевірені факти (номери 101/102/103/104, 112; правила ПДР) як **вхідні дані**, а не генерує їх із «пам'яті».
5. **Модерація — двоступенева:** автоматична перевірка (схема + фільтр неприйнятного змісту) → **затвердження вчителем**. Жоден згенерований урок не показується дитині без схвалення людини.
6. **Версіонування уроків.** Кожна редакція зберігається; вчитель може правити згенероване перед публікацією.

## 5. Озвучення: вимоги та рішення

### 5.1. Вимоги до TTS (обов'язкові)

1. **Природна українська** — нейронний голос, правильні наголоси, м'які приголосні; жодних «роботизованих» голосів для дітей.
2. **Приємний для дитини голос** — теплий, спокійний, бажано жіночий/дитячий регістр; єдиний голос-провідник для впізнаваності.
3. **Керованість** — темп (повільніше за дорослий контент), паузи, наголос (SSML або еквівалент).
4. **Пакетна генерація, не realtime.** Озвучення генерується **на етапі створення уроку** й зберігається як аудіофайли — дитячий пристрій лише програє готове аудіо. Це знімає вимоги до затримки, здешевлює й гарантує роботу на слабких пристроях/офлайн-кеші.
5. **Кешування.** Один раз згенерована фраза не генерується повторно.

### 5.2. Кандидати TTS API

Для української мови доступні кілька рівнів (порядок = рекомендована послідовність тестування):

- **ElevenLabs (Multilingual)** — <cite index="1-1">підтримує українську з природною емоційністю та передачею інтонацій</cite>; найкраща виразність для казкового/дитячого контенту, але найдорожчий: <cite index="3-1">від ~$60–120 за мільйон символів залежно від моделі</cite>. Виправдано завдяки пакетній генерації (обсяги малі).
- **Microsoft Azure TTS** — має нейронні українські голоси, повний SSML, помірна ціна. Сильний баланс якість/вартість.
- **Google Cloud TTS** — нейронні голоси uk-UA, SSML, стабільна інфраструктура.
- **Amazon Polly** — <cite index="4-1">нейронний TTS із SSML-контролем вимови, пауз і наголосів, оплата за фактом використання</cite>; перевірити актуальну якість українського голосу.
- **OpenAI TTS** — <cite index="3-1">багатомовний, від $15 за мільйон символів</cite>, але <cite index="3-1">голоси оптимізовані під англійську, що може знижувати якість для інших мов</cite> — тестувати обережно.

**Процедура вибору:** згенерувати той самий набір дитячих фраз (10–15 реплік: інструкція, похвала, м'яка корекція, правило безпеки) у 3–4 сервісах → сліпе прослуховування з дітьми/батьками/вчителем → зафіксувати вибір у цьому документі.

### 5.3. Розпізнавання мовлення (STT) для голосового керування

- **Базовий рівень:** Web Speech API (вбудований у браузери, безкоштовний) — для простих команд і коротких відповідей. Якість розпізнавання дитячої української **обов'язково перевірити прототипом на ранньому етапі** — це найризикованіший технічний елемент продукту.
- **Резервний рівень:** хмарний STT (Google Speech-to-Text, Azure Speech) — якщо вбудований не впорається з дитячим мовленням.
- **Принцип проектування:** голос — **додатковий** канал; кожна голосова дія має дублюватися дотиком. Продукт повністю функціональний без мікрофона.
- **Приватність:** аудіо дитини не зберігається; розпізнавання — лише в момент відповіді, за згодою батьків.

## 6. Інтерактивний ШІ-помічник (персонаж-провідник)

### 6.1. Роль і призначення

Дитину супроводжує **один постійний анімований персонаж-провідник** (той самий голос-провідник з розд. 5.1). Його завдання:

- **Пояснювати** — переформулювати інструкцію чи правило простіше, якщо дитина не зрозуміла («Поясни ще раз»).
- **Підказувати** — давати навідну підказку після помилки чи довгої паузи, не називаючи відповідь одразу.
- **Підтримувати** — хвалити за успіх, м'яко підбадьорювати після помилки, тримати місток у реальність («У справжньому житті роби так само»).
- **Відповідати на запитання дитини в межах теми уроку** — голосом або дотиком до персонажа.

Персонаж — це «обличчя» фідбеку з Бачення проекту: саме через нього дитина розуміє, що це навчання, а не просто гра.

### 6.2. Дворівнева архітектура відповідей

**Рівень 1 — скриптовані репліки (основний, ~90% взаємодій).**
Підказки, похвала, корекція, повторні пояснення генеруються **разом з уроком** тим самим LLM-конвеєром: для кожної сцени ШІ створює набір реплік помічника (2–3 рівні підказок, варіанти пояснень, реакції на типові помилки). Вони проходять ту саму модерацію вчителем і пакетне озвучення. На пристрої дитини це просто програвання готового аудіо — швидко, дешево, безпечно, працює офлайн.

**Рівень 2 — LLM у реальному часі (лише для вільних запитань дитини).**
Коли дитина ставить запитання, якого немає в скриптах, запит іде до LLM API — **у жорстких рамках**:

1. **Системний промпт-«клітка»:** помічник відповідає лише в межах теми поточного уроку та навчальних модулів; вік аудиторії 3–6 років; 1–2 короткі речення простою українською; завжди в характері персонажа.
2. **Заборонені теми → безпечна відмова.** На запитання поза межами (дорослі теми, особисті дані, будь-що недоречне) — одна стандартна лагідна відповідь: «Про це краще запитай маму, тата або вихователя. А ми повертаймося до уроку!» Помічник **ніколи не імпровізує** на чутливі теми.
3. **Вихідний фільтр.** Кожна відповідь LLM перед озвученням проходить автоматичну перевірку змісту (класифікатор/модераційний API). Не пройшла — програється безпечна стандартна репліка.
4. **Фактична дисципліна.** Факти безпеки помічник бере лише з перевіреної бази уроку (як у п. 4.2), не з «пам'яті» моделі.
5. **Обмеження діалогу.** Це не вільний чат-компаньйон: після 1–2 обмінів помічник повертає дитину до завдання. Помічник — частина уроку, а не заміна спілкування з людьми.

### 6.3. Технічні вимоги

- **Модель для рівня 2:** та сама LLM, що й для генерації уроків (Claude / GPT / Gemini), але **швидка/легка версія** (низька затримка, низька ціна) — відповіді короткі й прості, потужна модель не потрібна.
- **Озвучення відповідей рівня 2:** той самий TTS-сервіс і той самий голос, у потоковому або швидкому пакетному режимі; допустима затримка до ~2–3 с, яку персонаж «прикриває» анімацією задумливості.
- **Деградація без інтернету:** рівень 2 вимикається, працюють лише скриптовані репліки — урок повноцінний і без нього.
- **Журнал для дорослих:** усі вільні запитання дитини та відповіді помічника записуються в історію, доступну батькам і вчителю. Прозорість — обов'язкова.
- **Згода батьків** на функцію вільних запитань (рівень 2) — окремий перемикач у батьківському кабінеті; за замовчуванням можна вимкнути й лишити тільки скриптований рівень.

### 6.4. Педагогічні правила помічника

- Ніколи не соромить і не карає; помилка = «спробуймо ще раз, дивись…».
- Підказки — східчасті: спершу навідне запитання, потім вужча підказка, лише потім показ правильної дії.
- Не дає відповідь замість дитини, поки не вичерпані підказки.
- Регулярно проводить місток у реальність, особливо в модулі безпеки.
- Заохочує звертатися до живих дорослих — помічник не позиціонується як «друг замість людей».

## 7. Конвеєр створення уроку (підсумок)

```
Вчитель задає тему й параметри
        ↓
LLM (Claude / GPT / Gemini) генерує JSON-сценарій
за схемою сцен-шаблонів + перевіреними фактами
(включно з репліками помічника: підказки, пояснення, фідбек)
        ↓
Автоматична валідація (схема + фільтр змісту)
        ↓
Вчитель переглядає, редагує, затверджує
        ↓
Пакетна генерація озвучення (TTS) → аудіофайли
        ↓
Рушій рендерить урок з бібліотеки 2D-ассетів
        ↓
Дитина проходить урок (дотик + голос),
результати → історія для батьків і вчителя
```

## 8. Рішення, зафіксовані цим документом

1. Формат подання: **2D-вектор + повне озвучення + інтерактивні сцени-шаблони.** 3D не використовується.
2. ШІ генерує **дані уроку (JSON)**, а не графіку чи анімацію.
3. Тип ШІ для уроків: **LLM через API зі структурованим виводом** (Claude / GPT / Gemini — фінальний вибір за тестом української).
4. Озвучення: **пакетна генерація TTS при створенні уроку**, кешовані аудіофайли; кандидати — ElevenLabs, Azure, Google Cloud TTS, Polly (фінальний вибір за сліпим тестом).
5. Голосове керування: **Web Speech API як база, хмарний STT як резерв; голос завжди дублюється дотиком.**
6. Модерація людиною (вчителем) — **обов'язкова ланка** перед показом уроку дитині.
7. ШІ-помічник: **один постійний персонаж-провідник; дворівнева архітектура** — скриптовані репліки як основа (~90%), LLM у реальному часі лише для вільних запитань у жорстких рамках, з вихідним фільтром, журналом для батьків і можливістю вимкнення. Помічник — частина уроку, а **не вільний чат-компаньйон.**
