# Специфікація бібліотеки ассетів (MVP)

> **Статус:** зафіксоване рішення + технічне завдання на графіку. Ґрунтується на «Форматі подання уроків», «Вимогах до модуля Безпека», «UX-гайді» та `lesson-scenario.schema.json`.
>
> **Зафіксовані вибори:** провідник — **зайченя**; стиль — **плоский вектор (flat)**; виробництво — **гібрид**: персонаж і анімації — художник, фони й іконки — ШІ/сток + уніфікація.

---

## 1. Загальні вимоги до всіх ассетів

| Параметр | Вимога |
|---|---|
| Стиль | flat: чисті заливки, товстий контур єдиної ваги, без градієнтів/тіней/текстур |
| Формат статики | **SVG** (оптимізований: без растрових вкраплень, зайвих груп, «сирих» кривих) |
| Формат анімацій | **Lottie (JSON)**, вектор-only функції (без ефектів, що не підтримуються Lottie-плеєрами) |
| Іменування | id за патерном схеми `^[a-z0-9_]{2,50}$`; id = ім'я файлу (`rabbit_guide__joy__stand.svg`) — подвійне підкреслення розділяє варіанти |
| Палітра | всі кольори — **лише з токенів палітри** (розд. 2); жодних «своїх» відтінків — це основа майбутніх тем персоналізації |
| Реєстр | кожен ассет вноситься в `asset-registry.json` (id, тип, джерело, ліцензія, статус) — з нього експортується ASSET_REGISTRY для промпта генерації та перевірка бізнес-валідатора |
| Ліцензії | для стокових/ШІ-джерел — комерційна ліцензія без атрибуції у продукті; джерело і умови фіксуються в реєстрі. Стокові ассети обов'язково перемальовуються/уніфікуються під палітру (не «як є») |

## 2. Палітра (токени)

Базова тема `theme_default` — теплі, насичені, не «кислотні» (за UX-гайдом):

```
--c-primary      (акцент інтерактиву)     --c-bg-sky      (небо/фон верх)
--c-secondary    (другий акцент)          --c-bg-ground   (земля/фон низ)
--c-guide-main   (шерсть зайченяти)       --c-neutral-1..3 (сірі: дорога, контури)
--c-guide-accent (вушка/животик)          --c-success / --c-oops (лагідні!)
--c-danger-soft  (небезпечне у сортуванні — помітний, але не «кривавий» червоний)
```

Точні значення затверджуються з першим концептом персонажа; фіксуються в `/frontend` як CSS-змінні і в цьому документі.

## 3. Персонаж: зайченя (робоче ім'я — обрати до озвучення)

**Виробництво: художник.** Спершу — **модельний лист (character sheet)**: персонаж у фас/профіль/три чверті, пропорції, палітра, приклади емоцій. Затверджується до початку решти ассетів.

Характер у пластиці: доброзичливе, трохи наївне, «молодше за дитину»; довгі вуха — головний інструмент емоцій (підняті/опущені/одне зігнуте).

### 3.1. Статичні набори (SVG) — комбінації для схеми
Емоції (6): `neutral, joy, surprise, mild_fear, thinking, pride`
Пози (6): `stand, walk, stop, point, hold_phone, clap`

Не всі 36 комбінацій — лише вживані (орієнтир ~16–18):
| Поза \ Емоція | neutral | joy | surprise | mild_fear | thinking | pride |
|---|---|---|---|---|---|---|
| stand | ✔ | ✔ | ✔ | ✔ | ✔ | ✔ |
| walk | ✔ | ✔ | — | — | — | — |
| stop | ✔ | — | ✔ | ✔ | — | — |
| point | ✔ | ✔ | — | — | ✔ | — |
| hold_phone | ✔ | — | ✔ | ✔ | — | — |
| clap | — | ✔ | — | — | — | ✔ |

### 3.2. Анімації (Lottie) — значення `action` та системні
Зі схеми: `walk_to_road, stop, look_left_right, cross, pick_phone, dial, wave`
Наслідки (закритий список!): `guide_startled` (відскочив), `guide_oops` (ойкнув, вуха впали)
Системні (UX-гайд): `celebrate` (успіх/наліпка), `sleep_idle` (заставка «спить» після ліміту сесії), `idle_breathe` (очікування), `hint_gesture` (підказка рухом)

Разом: **13 Lottie-анімацій**, кожна 1.5–4 с, безшовний вхід/вихід зі stand.

## 4. Фони (SVG, ландшафтні 16:9 з запасом по краях)

**Виробництво: ШІ/сток + уніфікація палітри.**

| id | Опис | Уроки |
|---|---|---|
| `yard_day` | двір: майданчик, паркан, вдалині будинок | 1.1, 1.6, сортування |
| `street_edge` | край дороги без переходу, проїжджа частина | 1.1, 1.5, 1.6 |
| `street_crossing` | дорога із зеброю і місцем під світлофор | 1.2, 1.3, 1.4 |
| `room_phone` | дитяча кімната зі столиком і телефоном | 2.1–2.6 |

Вимога: центральна зона фону «спокійна» (там діють персонажі й варіанти вибору), деталізація — по краях.

## 5. Реквізит і об'єкти (SVG)

| Група | id (орієнтовно) |
|---|---|
| Дорога | `traffic_light_red`, `traffic_light_green`, `car_blue`, `car_red`, `car_yellow`, `ball_red` |
| Телефонія | `phone_handset`, `phone_screen_dial` |
| Служби — транспорт | `truck_fire`, `car_police`, `car_ambulance`, `car_gas` |
| Служби — люди | `person_firefighter`, `person_police`, `person_doctor`, `person_gas_worker` (нейтральні, доброзичливі) |
| Номери | `num_101`, `num_102`, `num_103`, `num_104`, `num_112` (великі, читабельні цифри — це і предмет вивчення) |

## 6. Іконки варіантів вибору та категорій (SVG, квадратні, великі)

Мова іконок: **дія, а не абстракція** (дитина «читає» картинку).

| id | Зміст |
|---|---|
| `icon_run_to_road` | фігурка біжить до дороги |
| `icon_call_adult` | дитина кличе дорослого (рука до рота) |
| `icon_wait_stand` | стоїть і чекає |
| `icon_go_walk` | іде |
| `icon_stop_hand` | долонька «стоп» |
| `icon_look_left_right` | голова дивиться обабіч |
| `icon_hold_hand` | тримає дорослого за руку |
| `icon_zebra_cross` | перехід по зебрі |
| `icon_cross_anywhere` | перехід поза зеброю (для «неправильного» варіанта — без драматизації) |
| `icon_dial_phone` | набирає номер |
| `icon_joke_phone` | «жартівливий» дзвінок (для патерна 2.5) |
| `cat_safe` / `cat_unsafe` | категорії сортування: зелене коло-усмішка / м'який знак уваги (НЕ череп, НЕ червоний хрест) |

*(Точний перелік поповнюється при генерації уроків: якщо ШІ потребує відсутньої іконки — бізнес-валідатор відхилить сценарій, і іконка або додається в реєстр, або сцена переробляється. Реєстр — джерело правди.)*

## 7. UI-ассети дитячого режиму (SVG/Lottie)

- Наліпки альбому (**за змістом вивченого**): `sticker_traffic_light`, `sticker_zebra`, `sticker_ball_lesson`, `sticker_101 … sticker_112`, `sticker_topic_road`, `sticker_topic_emergency` — ~10–12 шт.
- Полиця уроків: картка-рамка, «дихаюча» рамка поточного уроку (Lottie), слот наліпки.
- Прогрес сцен: «сліди лапок» (заповнені/порожні).
- Кнопки: `btn_next` (велика лапка/стрілка), `btn_repeat` (повторити — вушка), «дорослий кут» (маленька, навмисно нецікава дитині).
- Заставки: `splash_sleep` (зайченя спить — ліміт сесії), `splash_ask_adult` (немає мережі/кешу).
- Аватари дітей: 8–10 простих звіряток у тій самій палітрі.

## 8. План виробництва (гібрид)

| Етап | Хто | Результат |
|---|---|---|
| 1. Модельний лист зайченяти + палітра | художник | затвердження стилю ВСЬОГО проекту |
| 2. Статичні пози/емоції (~16–18 SVG) | художник | персонаж у сценах |
| 3. Lottie-анімації (13) | художник (або аніматор за модельним листом) | дії та системні стани |
| 4. Фони (4) | ШІ/сток → уніфікація палітри (можна силами команди у векторному редакторі) | сцени |
| 5. Реквізит + іконки + номери (~35–40 SVG) | ШІ/сток → уніфікація; критичні для змісту (номери, іконки дій) — перевірка «читабельності» на дітях | наповнення сцен |
| 6. UI-ассети + наліпки (~20) | мікс: наліпки краще художнику (це «нагорода», має бути смачною), решта — сток/ШІ | дитячий режим |

Порядок жорсткий: **нічого не малюється до затвердження модельного листа** — інакше стиль «роз'їдеться».

Орієнтовний обсяг MVP: **~90–100 ассетів**, з них художницьких ~35 (персонаж + анімації + наліпки) — саме там, де потрібні характер і консистентність.

---

## 9. Рішення, зафіксовані цим документом

1. Провідник — **зайченя**, flat-вектор; вуха — головний інструмент емоцій; модельний лист затверджується до всього іншого.
2. Формати: **SVG** (статика) + **Lottie** (13 анімацій); кольори — лише з токенів палітри (основа майбутніх тем).
3. Виробництво — **гібрид**: художник — персонаж/анімації/наліпки; ШІ/сток з уніфікацією — фони/реквізит/іконки; ліцензії фіксуються в реєстрі.
4. `asset-registry.json` — єдине джерело правди про ассети: з нього — реєстр для промпта генерації та перевірки валідатора.
5. Іконки варіантів — **мова дій**; для «неправильних» варіантів і категорії «небезпечно» — без страшних символів (без черепів, хрестів, блискавок).
